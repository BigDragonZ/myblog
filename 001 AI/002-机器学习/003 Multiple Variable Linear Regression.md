# 003 多变量线性回归 (Multi-Variable Linear Regression)

> **助教导语**：欢迎来到高维世界！从这一课开始，我们将彻底脱离初中数学的“平面坐标系”，正式迈入现代人工智能的底层基石——**线性代数（Linear Algebra）与高维空间**。

## 1. 场景还原：当视觉直觉失效时

想象你回到了波特兰的房产中介公司。这一次，你要处理的表格变复杂了：不仅有**房屋面积 ($x_1$)**，还新增了**卧室数量 ($x_2$)**、**楼层数 ($x_3$)** 和 **房屋房龄 ($x_4$)**。

- **任务 ($T$)**：根据这 4 个特征，预测房屋的最终售价。
    
- **经验 ($E$)**：包含 $x_1, x_2, x_3, x_4$ 以及目标价格 $y$ 的历史训练数据集。
    
- **性能指标 ($P$)**：预测价格与真实价格之间的均方误差 (MSE)。
    

### 1.1 收集数据：多维特征表格

我们的**经验 ($E$)** 扩展成了一张多维表格：

|**样本索引 (i)**|**面积(尺) x1​**|**卧室(个) x2​**|**楼层(层) x3​**|**房龄(年) x4​**|**真实价格(万美元) y**|
|---|---|---|---|---|---|
|1|2104|5|1|45|**460**|
|2|1416|3|2|40|**232**|
|3|852|2|1|35|**178**|

### 1.2 历史的困局：无法画图，怎么建模？

在单变量时代，我们可以在纸上画个散点图找规律。如果有 2 个特征（面积、卧室），我们可以想象在一个 3D 坐标系里，我们要找的是一块“平坦的硬纸板”（平面）去穿过它们。

**但是现在有 4 个特征！加上价格 $y$，总共有 5 个变量。** 人脑被锁死在三维世界里，无法画出 5 维空间的散点图。怎么破局？

**🔑 破局之道：信任代数，抛弃视觉。**

只要我们假定各个特征对房价的贡献是**独立且叠加**的，我们就可以无视维度的限制，直接把单变量公式 $h(x) = wx + b$ 暴力延长！

于是，多变量线性回归的**假设函数 (Hypothesis)** 诞生了：

$$h_{\mathbf{w},b}(\mathbf{x}) = w_1x_1 + w_2x_2 + w_3x_3 + w_4x_4 + b$$

- $w_1$ 就是面积的“基础单价”。
    
- $w_2$ 就是多一个卧室能“涨多少钱”。
	
- $w_3$ 就是楼层与价格的关系
    
- $w_4$ 是房龄的权重，因为房子越老越便宜，所以机器学出来的 $w_4$ 大概率是个负数。

## 2. 传统解法：不考虑向量化的“纯手工”实现

在不知道任何高级数学工具的前提下，一个初级程序员会怎么用 Python 实现多变量的梯度下降呢？

按照生命周期：初始化 $\rightarrow$ 计算代价 $\rightarrow$ 梯度下降 $\rightarrow$ 确认结果。我们将这几步用基础的 `for` 循环硬写出来。

### 2.1 明确数学计算逻辑 (白话版)

1. **算预测值**：用一个循环把 4 个特征和 4 个权重挨个相乘再相加。
    
2. **算代价函数**：将所有 $m$ 套房子的误差平方加起来，除以 $2m$。
    
    $$J(\mathbf{w},b) = \frac{1}{2m} \sum_{i=1}^{m} (h_{\mathbf{w},b}(\mathbf{x}^{(i)}) - y^{(i)})^2$$
    
3. **算梯度（下山的方向）**：对每个权重 $w_j$ 分别算坡度。注意公式里的小尾巴 $x_j^{(i)}$，它相当于一个“放大器”。
    
    $$dj\_dw_j = \frac{1}{m} \sum_{i=1}^{m} (h_{\mathbf{w},b}(\mathbf{x}^{(i)}) - y^{(i)}) \cdot x_j^{(i)}$$
    

### 2.2 纯手工 Python 嵌套循环代码

Python

```
# 1. 准备多变量数据 (经验 E)
X_train = [
    [2104, 5, 1, 45], # 房子 1
    [1416, 3, 2, 40], # 房子 2
    [852,  2, 1, 35]  # 房子 3
]
y_train = [460, 232, 178] # 真实价格

# 2. 随机初始化
w = [0.1, 0.2, 0.3, 0.4] 
b = 50.0

# 3. 笨办法：计算梯度 (寻找下山最陡的方向)
def compute_gradient_naive(X, y, w, b):
    m = len(X)    # 样本数量 (3)
    n = len(w)    # 特征数量 (4)
    
    dj_dw = [0] * n  # 存放 4 个权重的导数
    dj_db = 0        
    
    # 🚨 灾难开始：嵌套循环 🚨
    for i in range(m): # 第一层循环：遍历每一个房子
        # 1. 计算当前房子的预测价格 h(x)
        h_x = 0
        for j in range(n): # 第二层循环：遍历 4 个特征求和
            h_x += w[j] * X[i][j]
        h_x += b
        
        # 2. 计算误差
        error = h_x - y[i]
        
        # 3. 累加每个特征的梯度
        for j in range(n): # 第三层循环：分别计算 4 个特征的偏导数
            dj_dw[j] += error * X[i][j]
        dj_db += error
        
    # 求平均值
    for j in range(n):
        dj_dw[j] = dj_dw[j] / m
    dj_db = dj_db / m
    
    return dj_dw, dj_db

dj_dw, dj_db = compute_gradient_naive(X_train, y_train, w, b)
print(f"手工计算梯度: dj_dw = {dj_dw}")
```

### 2.3 痛点剖析：算力灾难

上面的代码能跑通，而且数学逻辑绝对正确。但在真实的 AI 工业界，**这行代码活不过第一轮 Code Review。**

**【苏格拉底式启发：为什么这套代码是灾难？】**

看看代码里的三层嵌套循环 `for epoch -> for i (样本) -> for j (特征)`。

假设我们在美团训练一个外卖推荐模型：

- $m$ (样本数) = 1 亿条用户行为。
    
- $n$ (特征数) = 1 万个用户标签。
    

如果用上面这套纯手工 Python 代码，CPU 需要串行执行 $1000 \times 100,000,000 \times 10,000 = \text{100 亿亿次}$ 计算！算完一次梯度下降，可能你的实习期都结束了。

**破局之道：线性代数的降维打击**

数学家发现，那一堆堆的 `for` 循环，本质上都在做同一件事：**对应位置相乘，再相加**。

这不就是线性代数里**矩阵点积 (Dot Product)** 的定义吗？

我们把所有的样本堆成一个巨大的矩阵 $\mathbf{X}$，把所有的权重打包成一个向量 $\mathbf{w}$。

- **手工版公式**：$h = \sum (w_j \cdot x_j) + b$
    
- **向量化公式**：$\mathbf{\hat{y}} = \mathbf{X} \cdot \mathbf{w} + b$
    

仅仅是用矩阵乘法替换了求和符号，我们就彻底唤醒了 CPU/GPU 底层的 **SIMD（单指令多数据流）** 物理电路。硬件不再是一个个算，而是**在同一个时钟周期内，同时向寄存器发射几百个特征进行并行相乘！**

## 3. 技术优化：引入向量化 (Vectorization)

为了消灭慢吞吞的 `for` 循环，我们需要引入线性代数的降维打击武器：**矩阵与向量**。

### 3.1 基础数学课：向量点积取代循环

**【物理直觉类比】**：

- **循环法**：顾客点菜，你拿计算器按：$w_1 \times x_1$（1盘鸡丁 $\times$ 30元）+ $w_2 \times x_2$（2碗米饭 $\times$ 5元）... 算得很慢。
    
- **向量法（点积）**：你把单价做成“清单 $\mathbf{w}$”，点菜量做成“菜单 $\mathbf{x}$”。把两样丢进超级点钞机（矩阵乘法），“滴”一声直接出总价！
    

**数学表达**：

我们把权重打包成列向量 $\mathbf{w}$，特征打包成列向量 $\mathbf{x}$。它们的**点积 (Dot Product)** 在数学定义上就是对应相乘再相加。

所以，冗长的公式瞬间变身为极其优雅的形态：

$$h_{\mathbf{w},b}(\mathbf{x}) = \mathbf{w}^T \mathbf{x} + b$$

_(注：$\mathbf{w}^T$ 是将列变行，为了满足矩阵乘法“前列数等于后行数”的规则)_

### 3.2 终极进化：矩阵取代样本遍历

刚才只是解决了一套房子的问题，如果有 $m$ 套房子呢？

我们把所有房子的特征像 Excel 一样叠成一个巨大的**特征矩阵 $\mathbf{X}$**（尺寸 $m \times n$）。

让矩阵 $\mathbf{X}$ 直接去乘列向量 $\mathbf{w}$。数学规律保证了，这一步操作会同时算出所有 $m$ 套房子的预测值！

**终极预测公式（无任何循环）**：

$$\mathbf{\hat{y}} = \mathbf{X}\mathbf{w} + b$$

### 3.3 向量化代码实战 (NumPy魔法)

让我们用底层矩阵霸主 `NumPy`，将刚才的嵌套循环重写为速度飙升百倍的工业代码：

Python

```
import numpy as np

# 1. 数据打包成 NumPy 矩阵和向量 (核心前提)
X_train = np.array([
    [2104, 5, 1, 45],
    [1416, 3, 2, 40],
    [852,  2, 1, 35]
])
y_train = np.array([460, 232, 178])
w = np.zeros(4) 
b = 0.0

# 2. 向量化计算
m = X_train.shape[0]

# 魔法 1：一句话算出所有房子的预测值！对应公式 Xw + b
predictions = np.dot(X_train, w) + b

# 魔法 2：一句话算出所有误差 (形状为 m x 1 的向量)
errors = predictions - y_train

# 魔法 3：一句话算出所有 4 个特征的梯度！(消灭所有 for 循环)
# 对应数学公式：(1/m) * X^T * errors
dj_dw = (1 / m) * np.dot(X_train.T, errors)
dj_db = (1 / m) * np.sum(errors)

print(f"向量化计算的梯度: dj_dw = {dj_dw}")
```

**【数学防坑指南：为什么要转置 `X.T`？】**

在算梯度时，`X` 是 $(m \times n)$，`errors` 是 $(m \times 1)$。

直接乘 `np.dot(X, errors)` 会报错，因为内部维度不匹配（$n \neq m$）。

我们将 `X` 转置成 $\mathbf{X}^T$（尺寸变成 $n \times m$），再去乘 $(m \times 1)$。不仅内部维度 $m$ 完美抵消，结果还刚好是一个 $(n \times 1)$ 的向量，严丝合缝地对应了我们要更新的 $n$ 个权重！这就是数学的暴力美学。

### 🧠 统治机器学习的三大终极矩阵公式：

1. **预测函数**：$\mathbf{\hat{y}} = \mathbf{X}\mathbf{w} + b$
    
2. **代价函数**：$J(\mathbf{w},b) = \frac{1}{2m} (\mathbf{\hat{y}} - \mathbf{y})^T (\mathbf{\hat{y}} - \mathbf{y})$
    
3. **梯度更新**：$\mathbf{w} = \mathbf{w} - \alpha \frac{1}{m} \mathbf{X}^T (\mathbf{\hat{y}} - \mathbf{y})$

## 5. 🛠️ 大厂工程视角 (Engineering Reality)

2. **工业 API 的黑盒**：在实际工作中，我们连上面的 NumPy 代码都不写，直接调用 `Scikit-Learn`：
    
    Python
    
    ```
    from sklearn.linear_model import LinearRegression
    ```

## 6. 数学证明

#### 第一阶段：消灭特征的循环 (计算单套房子的预测值)

假设我们现在只看**第 1 套房子**。它有 $n$ 个特征。

按照初中代数的写法，预测公式是：

$$h_{\mathbf{w},b}(\mathbf{x}^{(1)}) = w_1 x_1^{(1)} + w_2 x_2^{(1)} + \dots + w_n x_n^{(1)} + b$$

_(代码里，这需要一个遍历 $n$ 个特征的 `for` 循环)_

**【线性代数的降维魔法】**：

我们把所有的权重排成一列，叫作**列向量 $\mathbf{w}$**（尺寸是 $n \times 1$）。

把这套房子的所有特征也排成一列，叫作**列向量 $\mathbf{x}^{(1)}$**（尺寸是 $n \times 1$）。

$$\mathbf{w} = \begin{bmatrix} w_1 \\ w_2 \\ \vdots \\ w_n \end{bmatrix}, \quad \mathbf{x}^{(1)} = \begin{bmatrix} x_1^{(1)} \\ x_2^{(1)} \\ \vdots \\ x_n^{(1)} \end{bmatrix}$$

在线性代数中，**点积 (Dot Product)** 的定义，就是把对应位置的数字相乘，然后全部加起来。

为了让矩阵乘法的规则成立（前面矩阵的列数 = 后面矩阵的行数），我们把 $\mathbf{w}$ 躺平（转置，记为 $\mathbf{w}^T$），尺寸变成了 $1 \times n$。

然后让 $\mathbf{w}^T$ 去乘 $\mathbf{x}^{(1)}$：

$$\mathbf{w}^T \mathbf{x}^{(1)} = \begin{bmatrix} w_1 & w_2 & \dots & w_n \end{bmatrix} \begin{bmatrix} x_1^{(1)} \\ x_2^{(1)} \\ \vdots \\ x_n^{(1)} \end{bmatrix} = w_1 x_1^{(1)} + w_2 x_2^{(1)} + \dots + w_n x_n^{(1)}$$

你看！**右边的长串加法，完美坍缩成了左边的一个矩阵相乘 $\mathbf{w}^T \mathbf{x}^{(1)}$**。

所以单套房子的公式变成了：

$$h_{\mathbf{w},b}(\mathbf{x}^{(1)}) = \mathbf{w}^T \mathbf{x}^{(1)} + b$$

_(这消灭了内部遍历特征的那个 `for` 循环)_

---

#### 第二阶段：消灭样本的循环 (同时计算 $m$ 套房子)

现在，我们有 $m$ 套房子。我们难道要写个循环去算 $\mathbf{w}^T \mathbf{x}^{(1)}$, 然后算 $\mathbf{w}^T \mathbf{x}^{(2)}$ 吗？绝不！

**【构建特征矩阵 $\mathbf{X}$】**：

我们把所有房子的特征数据，像 Excel 表格一样堆叠起来。

每一行代表一套房子，每一列代表一个特征。这就构成了**设计矩阵 $\mathbf{X}$**（尺寸是 $m \times n$）。

$$\mathbf{X} = \begin{bmatrix} --- & (\mathbf{x}^{(1)})^T & --- \\ --- & (\mathbf{x}^{(2)})^T & --- \\ & \vdots & \\ --- & (\mathbf{x}^{(m)})^T & --- \end{bmatrix} = \begin{bmatrix} x_1^{(1)} & x_2^{(1)} & \dots & x_n^{(1)} \\ x_1^{(2)} & x_2^{(2)} & \dots & x_n^{(2)} \\ \vdots & \vdots & \ddots & \vdots \\ x_1^{(m)} & x_2^{(m)} & \dots & x_n^{(m)} \end{bmatrix}$$

**【奇迹时刻：矩阵 $\mathbf{X}$ 乘以向量 $\mathbf{w}$】**：

我们让尺寸为 $(m \times n)$ 的矩阵 $\mathbf{X}$，去乘以尺寸为 $(n \times 1)$ 的列向量 $\mathbf{w}$。

根据矩阵乘法规则（拿前面矩阵的每一行，去和后面矩阵的列做点积），我们得到了一个极度优美的结果：一个 $(m \times 1)$ 的列向量！

$$\mathbf{X} \mathbf{w} = \begin{bmatrix} --- (\mathbf{x}^{(1)})^T --- \\ --- (\mathbf{x}^{(2)})^T --- \\ \vdots \\ --- (\mathbf{x}^{(m)})^T --- \end{bmatrix} \begin{bmatrix} w_1 \\ w_2 \\ \vdots \\ w_n \end{bmatrix} = \begin{bmatrix} \mathbf{w}^T\mathbf{x}^{(1)} \\ \mathbf{w}^T\mathbf{x}^{(2)} \\ \vdots \\ \mathbf{w}^T\mathbf{x}^{(m)} \end{bmatrix}$$

你看这个结果向量里的每一项，不正好就是**每套房子的预测价格（不含 $b$）**吗？！

最后，利用 Python 的广播机制（Broadcasting），我们直接给这个列向量加上常数 $b$。

**最终推导结论**：

包含所有 $m$ 套房子的预测值列向量 $\mathbf{\hat{y}}$，其完全无循环的纯数学表达就是：

$$\mathbf{\hat{y}} = \mathbf{X}\mathbf{w} + b$$
