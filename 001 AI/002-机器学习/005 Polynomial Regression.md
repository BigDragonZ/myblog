多项式回归：让“直线”学会转弯

> **导语**：在真实世界中，变量间的关系很少是完美的直线。比如房价，面积翻倍价格未必翻倍；再比如生物生长，初期极快后期变缓。我们将学习如何不改变线性回归的“骨架”，只通过“特征炼金术”来拟合曲线。

## 🎯 核心概念与物理直觉 (Concept & Intuition)

**场景还原**：

你发现波特兰的豪宅市场有个规律：小房子的单价很高，但当面积超过 5000 平方英尺后，价格增长开始“疲软”甚至持平（因为维护成本太高）。如果你坚持用那把**硬邦邦的直尺**（线性模型），你会发现：面积小时预测偏低，面积大时预测偏高。

**物理类比**：

多项式回归就像是**“高维投影”**。想象你在地面（一维）上拉一根紧绷的绳子，它只能是直的。但如果你把绳子的某些部分拎到空中（增加 $x^2, x^3$ 维度），从地面俯瞰，这根绳子的投影就变成了优美的曲线。

### TPE 三要素拆解：

- **任务 (T)**：给定房屋面积 $x$，预测具有非线性趋势的价格 $y$。
    
- **经验 (E)**：包含原始特征 $x$ 及其衍生特征（如 $x^2, \sqrt{x}$）的增强数据集。
    
- **性能指标 (P)**：均方误差 $J(\mathbf{w}, b)$。

## 🔗 维度跃迁 (Dimensional Upgrade)

为了处理曲线，我们需要把问题从“一维直线”升级到“高维特征空间”。

|**维度**|**模型表达式**|**几何直觉**|**特征工程**|
|---|---|---|---|
|**1 维 (旧知)**|$y = wx + b$|一条死板的直线|只有原始面积 $x$|
|**2 维 (新知)**|$y = w_1x + w_2x^2 + b$|一条平滑的抛物线|创造新特征：面积的平方|
|**3 维 (进阶)**|$y = w_1x + w_2x^2 + w_3x^3 + b$|拥有两个弯折的曲线|创造新特征：面积的立方|

**核心痛点**：为什么不直接发明一种“曲线公式”？

**助教启发**：如果我们定义 $x_1 = x, x_2 = x^2, x_3 = x^3$，那么公式就变成了 $y = w_1x_1 + w_2x_2 + w_3x_3 + b$。瞧！这不就是你上节课刚学过的**多变量线性回归**吗？我们成功把非线性问题“骗”回了线性框架里，这样你辛苦推导的梯度下降算法依然生效！

## 🧮 数学推演与向量化表达 (Math & Vectorization)

在多项式回归中，**特征缩放 (Feature Scaling)** 不再是可选，而是**保命符**。

**维度爆炸问题**：

如果面积 $x = 1000$，那么：

- $x$ 特征 = $10^3$
    
- $x^2$ 特征 = $10^6$
    
- $x^3$ 特征 = $10^9$
    

代价函数 $J$ 的等高线会变成一个**极其扁长的“发丝级”椭圆**。如果不缩放，你的梯度下降会在这里反复横跳，直到程序崩溃。

**矩阵表达**：

我们将原始数据矩阵 $\mathbf{X}$ 转化为增强矩阵 $\mathbf{X}_{poly}$：

$$\mathbf{X}_{poly} = \begin{bmatrix} 1 & x^{(1)} & (x^{(1)})^2 \\ 1 & x^{(2)} & (x^{(2)})^2 \\ \vdots & \vdots & \vdots \end{bmatrix}$$

## 💻 代码实战 (底层逻辑 vs 工业 API)

### 2.1 底层逻辑：用 NumPy “炼金”

我们要手动创造出 $x^2$ 特征，然后喂给之前的线性回归模型。

Python

```
import numpy as np

# 1. 原始数据 (1D)
x = np.array([1, 2, 3, 4, 5])
y = np.array([1.2, 4.5, 9.1, 16.2, 24.8]) # 明显接近平方关系

# 2. 特征工程：手动构造多项式矩阵 [x, x^2]
X_poly = np.c_[x, x**2] 

# 3. 特征缩放 (关键！)
X_scaled = (X_poly - np.mean(X_poly, axis=0)) / np.std(X_poly, axis=0)

# 4. 剩下的交给之前的线性回归代码即可
# w, b = gradient_descent(X_scaled, y, alpha=0.1, iters=1000)
```

### 2.2 工业级实现：Scikit-Learn Pipeline

在大厂开发中，我们使用 `Pipeline` 来确保特征转换和预测的严谨性。

Python

```
from sklearn.preprocessing import PolynomialFeatures, StandardScaler
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline

# 封装一个“多项式回归”流水线
model = Pipeline([
    ("poly_features", PolynomialFeatures(degree=2, include_bias=False)), # 自动生成 x^2
    ("scaler", StandardScaler()),                                      # 自动 Z-score 缩放
    ("lin_reg", LinearRegression())                                    # 线性回归核心
])

model.fit(x.reshape(-1, 1), y)
print(f"预测 6 平方英尺的价格: {model.predict([[6]])}")
```

## 🛠️ 大厂工程视角 (Engineering Reality)

在处理像 Google 搜索排名或 Meta 推荐系统时，我们面临的真实挑战：

1. **特征组合 (Feature Crosses)**：现实中不仅有 $x^2$，还有“面积 $\times$ 邮政编码”。这种跨特征组合才是大厂提升精度的核武器。
    
2. **过拟合 (Overfitting) 陷阱**：如果你用 10 阶多项式去拟合 5 个点，曲线会穿过每个点，但它会变得异常扭曲。这在工程上被称为“记住了数据”而不是“学会了规律”。
    
3. **计算代价**：高阶多项式会让特征数量指数级增长， TB 级数据下内存会直接溢出。因此，工业界更倾向于使用**分段线性回归 (Splines)** 或者更复杂的**树模型**。
