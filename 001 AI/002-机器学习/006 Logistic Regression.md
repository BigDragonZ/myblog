逻辑回归 (Logistic Regression)：分类的觉醒

> **导语**：当我们需要机器做出“是或否”的决策时，传统的线性回归开始显露疲态。我们将通过一次失败的尝试，引出分类算法的真王。

## 🎯 1. 场景还原：强行复用“直尺”的代价

想象你正在为一家医院开发**肿瘤识别系统**。

- **特征 ($x$)**：肿瘤的大小。
    
- **标签 ($y$)**：0 代表良性，1 代表恶性。
    

### 1.1 方案 A：继续用线性回归 (Linear Regression)

你拿出了之前的“直尺”模型：$f_{w,b}(x) = wx + b$。

- **决策逻辑**：如果预测值 $\ge 0.5$，判定为“恶性”；否则为“良性”。
    
- **初步运行**：看起来不错，直线完美地划分了两类数据。
    

### 1.2 遭遇滑铁卢：离群点 (Outliers) 危机

突然，医院送来了一个**极大的恶性肿瘤样本**。为了减小均方误差 (MSE)，线性回归的直线必须“迁就”这个远处的点，导致整条直线斜率变缓。

- **后果**：由于直线变平，原本一些明显的恶性肿瘤，现在的预测值降到了 0.5 以下，被误诊为“良性”！
    

> **失败总结**：分类问题需要的不是“无限延伸的线”，而是一道**稳健的屏障**。预测值必须被严格限制在 $[0, 1]$ 之间。

---

## 🔗 2. 维度跃迁：引入 Sigmoid 激活函数

为了解决输出溢出的问题，我们要给线性公式套上一层“紧箍咒”——**Sigmoid 函数**。

### 2.1 物理建模：S型曲线

我们将线性输出 $z = \mathbf{w}^T\mathbf{x} + b$ 喂给 Sigmoid 函数 $g(z)$：

$$g(z) = \frac{1}{1 + e^{-z}}$$

### 2.2 逻辑映射

- 当 $z$ 很大时，$g(z) \approx 1$（极大概率为恶性）。
    
- 当 $z$ 很小时，$g(z) \approx 0$（极大概率为良性）。
    
- **决策边界 (Decision Boundary)**：当 $\mathbf{w}^T\mathbf{x} + b = 0$ 时，我们正好处在 50% 的概率分界线上。

## 🧮 3. 数学推演：为什么 MSE 再次失效？

在回归任务中，我们习惯用均方误差 $J(w,b) = \frac{1}{2m}\sum(f-y)^2$。但当你把 Sigmoid 带入这个公式后，灾难发生了。

### 3.1 撞南墙：非凸 (Non-Convex) 陷阱

由于 Sigmoid 的非线性，此时的代价函数图像不再是一个平滑的碗，而是变成了**“月球表面”**。

- **问题**：充满了局部极小值点。
    
- **后果**：梯度下降会随机卡在某个坑里，无法找到全局最优解。
    

[Image comparing Convex cost function of Linear Regression vs Non-convex cost function of Logistic Regression with MSE]

### 3.2 进化：交叉熵损失 (Cross-Entropy)

为了把“月球表面”重新填平为“大碗”，我们需要新的打分规则：

- 如果真实 $y=1$，预测 $f$ 越接近 0，惩罚应无限大。
    
- 如果真实 $y=0$，预测 $f$ 越接近 1，惩罚应无限大。
    

由此得出**对数损失函数**：

$$Loss(f, y) = -y\log(f) - (1-y)\log(1-f)$$

## 💻 4. 代码实战：矢量化实现

### 4.1 NumPy 底层逻辑 (复用梯度下降)

神奇的是，虽然代价函数变了，但经过微积分推导，其**梯度公式的矢量化形式**竟然和线性回归保持了惊人的一致！

Python

```
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

# 梯度计算：完全复用多变量线性回归的矢量化形式
# z = Xw + b
# f = sigmoid(z)
# error = f - y
# gradient = (1/m) * X.T @ error
```

### 4.2 工业级实现 (Scikit-Learn)

Python

```
from sklearn.linear_model import LogisticRegression

# 工业界标准：处理分类问题
model = LogisticRegression()
model.fit(X_train, y_train)

# 输出预测概率，而不仅仅是标签
prob = model.predict_proba(X_test) 
```

## 🛠️ 5. 大厂工程视角 (Engineering Reality)

1. **概率阈值可调**：大厂不会死守 0.5 这个阈值。在**反欺诈**场景下，宁可错杀一千不可放过一个，我们可能会把阈值调到 0.1；而在**垃圾邮件过滤**中，为了不误删重要邮件，阈值可能调到 0.9。
    
2. **特征饱和 (Saturation)**：如果特征没做标准化，某个 $x$ 很大，会导致 $z$ 极其巨大，Sigmoid 进入平坦区。此时**梯度几乎为 0**，模型会停止学习。这就是为什么**特征缩放**在逻辑回归中同样重要！
