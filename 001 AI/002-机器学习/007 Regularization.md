你好！非常有远见的决定。把正则化（Regularization）独立出来，不仅是因为它能解决多项式的“弯曲”问题，更因为它在**回归**和**分类**中起到的作用逻辑是高度统一的。

通过这一章，我们将建立起机器学习中最重要的直觉：**通过在代价函数中添加“惩罚项”，强制让模型保持“简单”。**

# 005.5 正则化 (Regularization)：回归与分类的终极天平

> **导语**：模型就像一棵树。如果不加修剪，它会为了接触阳光而向四面八方疯狂生长（过拟合）。正则化就是那把剪刀，它通过惩罚过大的权重，让模型长得更健壮、更平滑。

## 🎯 1. 场景还原：当“聪明”过了头

无论是预测房价（回归）还是识别肿瘤（分类），当我们引入高阶特征（如 $x^2, x^3, \dots, x^{10}$）后，模型会产生一种“执念”：它试图穿过每一个训练数据点。

- **回归中的过拟合**：房价曲线不再平滑，而是为了贴合每一个噪点变得跌宕起伏。
    
- **分类中的过拟合**：判定边界不再是圆润的弧线，而是像锯齿一样扭曲。
    

**物理直觉**：

过拟合往往伴随着**参数 $w$ 的数值爆炸**。模型为了让曲线剧烈抖动，必须给某些特征施加巨大的权重。正则化的核心思想就是：**如果你想拥有巨大的权重，你必须付出代价（增大 Cost）**。

---

## 🔗 2. 维度跃迁：统一的惩罚机制

正则化对回归和分类的影响是跨维度的。

|**特性**|**无正则化 (Overfit)**|**有正则化 (Robust)**|**逻辑映射**|
|---|---|---|---|
|**权重 $w$ 的规模**|极大（数值可能达到成千上万）|较小（通常在 1 左右或更小）|参数规模与模型复杂度成正比|
|**决策/拟合形态**|极度扭曲、锯齿状、对噪音敏感|平滑、自然、泛化能力强|简单即是美（奥卡姆剃刀）|
|**算法鲁棒性**|遇到新数据直接“翻车”|遇到新数据表现稳定|降低了对个别样本的依赖|

## 🧮 3. 数学推演：代价函数的“紧箍咒”

我们将正则化项（Regularization Term）直接挂在原来的代价函数后面。

### 3.1 线性回归的正则化 (L2 / Ridge)

$$J(\mathbf{w},b) = \underbrace{\frac{1}{2m} \sum_{i=1}^{m} (f - y)^2}_{\text{误差项}} + \underbrace{\frac{\lambda}{2m} \sum_{j=1}^{n} w_j^2}_{\text{正则化项}}$$

### 3.2 逻辑回归的正则化 (L2)

$$J(\mathbf{w},b) = \underbrace{-\frac{1}{m} \sum_{i=1}^{m} [y\log(f) + (1-y)\log(1-f)]}_{\text{误差项}} + \underbrace{\frac{\lambda}{2m} \sum_{j=1}^{n} w_j^2}_{\text{正则化项}}$$

- **$\lambda$ (Lambda)**：惩罚强度拨盘。
    
    - $\lambda = 0$：无正则化，模型野蛮生长。
        
    - $\lambda$ 巨大：权重 $w$ 全部趋向 0，模型变成一条死板的直线（欠拟合）。
        

**注意**：我们不惩罚偏置项 $b$。因为 $b$ 只负责上下移动或左右偏移，不控制曲线的“抖动”频率。

## 💻 4. 代码实战：梯度下降的“权重衰减”

最神奇的地方在于，当你对带正则化的 $J$ 求导时，权重更新公式发生了统一的变化。

### 4.1 手动实现逻辑 (NumPy)

Python

```
# 逻辑回归与线性回归共用的正则化梯度更新逻辑
# 每一轮迭代，w 都会先“缩小”一点，再去减去梯度
# alpha 是学习率，lambda_ 是正则化参数

# 1. 计算原始梯度 (误差驱动)
dj_dw_mse = (1/m) * np.dot(X.T, (f - y))

# 2. 计算正则化带来的额外梯度 (规模惩罚)
dj_dw_reg = (lambda_ / m) * w

# 3. 合并更新
w = w - alpha * (dj_dw_mse + dj_dw_reg)
```

### 4.2 工业级 API 对比

Python

```
from sklearn.linear_model import Ridge, LogisticRegression

# 线性回归的正则化版
reg_linear = Ridge(alpha=1.0) # alpha 就是这里的 lambda

# 逻辑回归的正则化版 (注意：sklearn 中用 C = 1/lambda)
reg_logistic = LogisticRegression(C=1.0, penalty='l2') 
```

## 🛠️ 5. 大厂工程视角 (Engineering Reality)

1. **L1 (Lasso) vs L2 (Ridge)**：
    
    - **L2 (平方和)**：让 $w$ 整体变小，但不会变成 0。适合“所有特征都有点用”的场景。
        
    - **L1 (绝对值和)**：会将不重要的 $w$ 彻底压缩为 **0**。在处理有数万个特征的**稀疏数据**（如点击率预测）时，大厂必选 L1，因为它能顺便把无效特征给删了。
        
2. **特征缩放的生死线**：正则化项直接惩罚 $w$ 的数值。如果你没做特征缩放，数值大的特征（如月收入）对应的 $w$ 本身就很小，正则化对它的影响微乎其微；而数值小的特征（如年龄）对应的 $w$ 很大，会被正则化“误杀”。**不缩放，正则化就是乱杀无辜。**
