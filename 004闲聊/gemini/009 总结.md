### 第一阶段：基础设施与基建底座 (Infrastructure)

这是所有代码运行的土壤，必须最先打通。

- **跨端控制与同步**：在 Mac 上安装 Claude Code 并开启 `claude remote-control` 模式，以便直接操作 Windows 环境。配置 Mac 到 WSL2 的 SSH (端口 2222) 免密登录，并编写 `rsync` 增量同步脚本。
    
- **WSL2 纯净环境**：在 WSL2 (Ubuntu) 中安装 Node.js 22、Docker（用于隔离技能插件），并使用 `uv` 初始化 Python 项目结构。
    
- **依赖管理**：通过 `pyproject.toml` 或 `requirements.txt` 统一管理 `akshare`、`duckdb`、`langchain`、`chromadb` 等量化与 AI 依赖。
    

### 第二阶段：数据血脉与 RAG 知识库 (Data & Knowledge)

将外部的经验和数据转化为你可以绝对掌控的本地资产。

- **知识库爬取与清洗**：封装 `ZhihuArchiverSkill` 脚本，将你下载的 61 篇 Mr. Dang 知乎文章转化为纯净的 Markdown 格式，并将图片存入本地 `assets` 目录，防盗链且方便 AI 识别。
    
- **领域分离向量库**：利用 RTX 40 系显卡部署本地 Embedding 模型，在 ChromaDB 中建立独立的 Collection（如区分数学、AI、经济学），通过语义路由（Semantic Router）避免检索干扰。
    
- **结构化市场数据**：利用 `yfinance` 或 `akshare` 获取全市场实时/历史行情及财务报表，并规划数据自动增量更新脚本。
    

### 第三阶段：Human-in-the-loop 决策大脑 (Agent & Skills)

从传统的 If-Else 代码升级为“智能体+工具箱”的现代架构。

- **投资价值观配置**：在 Mac 端编写 `valuation_rules.yaml` 和 `stock_pool.yaml`，固化你对 ROE、PE、FCF 等价值投资指标的底线要求。
    
- **原子化技能开发**：将所有能力封装为独立的 Python 函数（Skills），并带有清晰的 Docstring 供 AI 阅读。例如 `skill_get_financials` (获取财报) 和 `skill_query_knowledge_base` (检索护城河原则)。
    
- **MCP Server 包装**：使用 FastMCP 将上述 Python 脚本打包为标准服务，以便后续 OpenClaw 直接调用。
    

### 第四阶段：OpenClaw 部署与 24 小时监控 (Execution)

让你的助手长出“嘴巴”和“耳朵”，时刻盯着市场。

- **官方源码部署**：在 WSL2 中通过 GitHub 官方渠道克隆并启动 OpenClaw，避免使用不明来源的安装包以防木马。
    
- **注入灵魂设定**：编写 `SOUL.md` 文件，定义 Agent 为“保守的价值投资助理”，规定其必须先查知识库再做决策的铁律。
    
- **实时告警通道**：配置 Telegram Bot 或飞书 Webhook。编写守护进程 `price_watcher.py`，一旦价格跌入你的理想区间，立刻通过 OpenClaw 强穿透推送到你的手机。
    

### 第五阶段：风控与灾备 (Risk Management)

投资系统的底线是不出错、不丢失数据。

- **数据冷备份**：编写 `backup_db.sh` 脚本，利用 Linux 的 `cron` 定时任务，每天自动打包向量数据库和配置文件同步到云端。
    
- **防呆与熔断机制**：确保代码中包含 API 限流重试机制；严格限制 AI 权限（仅提供只读 API，无交易权限）；坚守“AI 分析仅作参考，人类把控最终决策”的原则。

你提到如果问题不准确希望得到优化指导。为了配合 Claude Code 的工作流，建议你在后续开发时**避免“宏大叙事”，改为“模块化指令”**。

- **✅ 优化的提问**：“作为第一步，请在当前的 WSL2 目录中利用 `uv` 初始化项目，并帮我编写第三阶段中的 `skill_get_financials.py`。要求：使用 `yfinance` 获取指定股票的 PE 和 ROE，并返回字典格式。”
