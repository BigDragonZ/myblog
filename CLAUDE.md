# MyBlog - 个人知识管理系统

## 项目概述

这是一个基于 [Obsidian](https://obsidian.md/) 构建的个人知识管理库（PKM），用于记录学习笔记、投资分析、项目规划和工具使用等内容。

- **工具**: Obsidian + Git 版本控制
- **同步**: GitHub 远程仓库
- **命名**: 使用中文序号 + 空格 + 分类名的格式

## 命名规范

### 文件夹命名
- **格式**: `三位数字` + `空格` + `中文分类名`
- **示例**: `001 AI`, `002 投资`, `999life`

### 文件命名
- **学习笔记**: `三位数字` + `空格` + `主题.md`
- **示例**: `001 音标.md`, `001 introduction.md`

### 特殊目录
- `999/`: 临时资源，待分类或归档
- `999life/`: 生活方法、个人成长
- `z_img/`: 所有图片资源集中存放

## 常用操作

### 拉取更新
```bash
git pull origin master
```

### 提交更改
```bash
git add .
git commit -m "描述修改内容"
git push origin master
```

### 暂存本地修改（拉取前）
```bash
git stash push -m "本地修改"
git pull origin master
git stash pop
```

## 工作流建议

1. **定期同步**: 每天开始工作前执行 `git pull`
2. **及时提交**: 完成一个主题或功能后及时提交
3. **清晰命名**: 提交信息要简明扼要，描述修改内容
4. **分类整理**: 新建笔记时放入正确分类，不确定时放 `999/`
5. **图片管理**: 所有图片统一放入 `z_img/`，使用相对路径引用

---

## 注意事项

- **Obsidian 配置**: `.obsidian/workspace.json` 会频繁变化，无需手动提交
- **敏感信息**: 不要在笔记中记录密码、Token 等敏感信息（使用 `.gitignore` 保护）
- **大文件**: 不要提交视频、大型数据集，使用外部存储或链接
- **CRLF 警告**: Windows 环境下 Git 会自动处理换行符，忽略相关警告即可
